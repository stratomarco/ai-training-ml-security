# BrokenPilot Capstone Final Rubric

## Total score: 100 points

| Category | Points |
|---|---:|
| System understanding and architecture | 15 |
| Threat model quality | 20 |
| Findings and evidence | 25 |
| Mitigation and secure design | 20 |
| Residual risk and prioritization | 10 |
| Communication and presentation | 10 |

## 1. System understanding and architecture — 15 points

| Score | Description |
|---|---|
| 13–15 | Clearly explains components, data flows, trust boundaries, users, tools, RAG, memory, and workflow impact. |
| 9–12 | Covers most major components but misses some trust boundaries or data flows. |
| 5–8 | Basic system description with limited architecture reasoning. |
| 0–4 | Unclear system understanding. |

## 2. Threat model quality — 20 points

| Score | Description |
|---|---|
| 17–20 | Strong asset inventory, realistic personas, entry points, abuse cases, trust boundaries, and prioritized risks. |
| 13–16 | Good threat model with minor gaps. |
| 8–12 | Basic threat model but limited abuse-case depth. |
| 0–7 | Mostly generic or incomplete. |

## 3. Findings and evidence — 25 points

| Score | Description |
|---|---|
| 21–25 | Findings are specific, evidence-backed, safe, impact-focused, and tied to failed controls. |
| 16–20 | Good findings with some evidence or impact gaps. |
| 10–15 | Findings are plausible but too generic or payload-focused. |
| 0–9 | Findings are unclear, unsupported, or unsafe. |

## 4. Mitigation and secure design — 20 points

| Score | Description |
|---|---|
| 17–20 | Mitigations are specific, implementable, risk-based, and placed in the correct system layer. |
| 13–16 | Good mitigations but some are vague or incomplete. |
| 8–12 | Mostly generic controls with limited implementation detail. |
| 0–7 | Mitigations rely mainly on prompting or are not actionable. |

## 5. Residual risk and prioritization — 10 points

| Score | Description |
|---|---|
| 9–10 | Clear priority model, owners, validation tests, and honest residual-risk statement. |
| 6–8 | Good prioritization with some residual-risk gaps. |
| 3–5 | Basic priority list but weak trade-off reasoning. |
| 0–2 | No meaningful prioritization or residual-risk discussion. |

## 6. Communication and presentation — 10 points

| Score | Description |
|---|---|
| 9–10 | Clear executive summary, concise top risks, strong leadership readout, and actionable language. |
| 6–8 | Understandable but could be sharper or better prioritized. |
| 3–5 | Too technical, too vague, or poorly organized. |
| 0–2 | Hard to understand or not decision-oriented. |

## Bonus guidance

Do not award high scores for exploit creativity alone.

Reward:

- security judgment;
- root-cause analysis;
- control placement;
- clarity;
- realistic engineering trade-offs;
- residual-risk honesty.
