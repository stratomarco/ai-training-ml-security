# Module Cut and Expand Guide

This guide helps instructors decide what to cut or expand when time changes.

## Always keep

Every delivery format should preserve these ideas:

1. The model is not the security boundary.
2. Data, prompts, retrieved context, tools, and memory are attack surfaces.
3. Security controls must exist outside the model.
4. Good AI security balances security, usability, and developer velocity.
5. Findings need root cause, impact, and remediation, not just screenshots.

## Cut first

When short on time, cut:

- Long taxonomy comparisons
- Detailed reference history
- Secondary reading lists
- Full quiz review
- Less relevant module-specific examples

## Cut last

Do not cut unless absolutely necessary:

- Threat boundaries
- Abuse cases
- Concrete attack path
- Mitigation design
- Residual risk discussion

## Expand first

When more time is available, expand:

- Hands-on labs
- Group threat modeling
- Fix/control design
- Report writing
- Executive communication
- Instructor debrief
