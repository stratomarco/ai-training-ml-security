# Code of Conduct

## Purpose

This project exists to teach practical ML Security with professionalism, respect, and a defensive security mindset.

## Expected behavior

Participants are expected to:

- Treat others with respect.
- Keep discussions technical and constructive.
- Avoid harassment, personal attacks, or discriminatory language.
- Use labs only in authorized environments.
- Respect third-party licenses and intellectual property.
- Share feedback in a way that improves the course.

## Unacceptable behavior

Unacceptable behavior includes:

- Harassment or intimidation.
- Publishing private information without consent.
- Encouraging unauthorized attacks.
- Sharing real secrets, credentials, or private data in issues or pull requests.
- Turning controlled lab material into instructions for attacking real systems.

## Enforcement

Maintainers may remove issues, comments, pull requests, or contributors that violate this code of conduct.

## Reporting

If the repository is public, report conduct issues to the repository owner or maintainer through the available private contact path.
