# Source of Truth Policy

The repository uses one canonical location for each type of course content.

## Canonical content locations

| Content type | Canonical location |
|---|---|
| Course modules | `modules/` |
| Labs and runnable prototypes | `labs/` |
| Templates | `course-templates/` |
| Instructor material | `instructor/` |
| Assessments and rubrics | `assessments/` |
| Website-only landing/setup/release pages | `docs/` |

Do not hand-maintain copies of modules, labs, templates, instructor material, or assessments under `docs/`.

## MkDocs generation

MkDocs builds from `.mkdocs-src/`, which is generated by:

```powershell
python scripts\sync_mkdocs_content.py
```

or on macOS/Linux:

```bash
python3 scripts/sync_mkdocs_content.py
```

The generated `.mkdocs-src/` directory is ignored by Git.

## Cleanup before committing

If a stale duplicate tree appears, remove it before committing:

```powershell
.\scripts\cleanup_duplicate_content.ps1
python scripts\sync_mkdocs_content.py
python scripts\check_repo_structure.py
```

or on macOS/Linux:

```bash
./scripts/cleanup_duplicate_content.sh
python3 scripts/sync_mkdocs_content.py
python3 scripts/check_repo_structure.py
```

## Why this matters

The course previously had hand-maintained copies under both root-level directories and `docs/`. That created drift risk: the website could show stale material while the root course content continued to evolve.

The policy now is simple:

> Edit canonical course files in the root-level content directories. Generate website source before preview or deployment.
