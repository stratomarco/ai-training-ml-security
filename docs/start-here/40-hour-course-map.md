# 40-Hour Course Map

This is the recommended one-week path through the course.

The course contains more material than an instructor should teach live. The live week focuses on concepts, validated labs, control design, evidence quality, and the BrokenPilot capstone. Deeper readings and optional quizzes are used for self-study.

## Week structure

| Day | Theme | Main modules | Main output |
|---|---|---|---|
| Day 1 | Foundations and architecture | 01, 02, 03, 04 | System context, trust boundaries, abuse cases |
| Day 2 | LLM and RAG security | 05, 06 | Evidence log and mitigation design |
| Day 3 | Agent security and supply chain | 07, 08 | Tool control evidence or executive risk memo |
| Day 4 | Privacy, robustness, and reporting | 09, 10, 11 | Strong rewritten finding |
| Day 5 | BrokenPilot capstone | 12 | Final report and presentation |

## What students should prioritize

Prioritize:

- understanding the security boundary,
- collecting evidence,
- designing enforceable controls,
- validating fixes,
- explaining residual risk,
- producing decision-grade reports.

Do not prioritize:

- collecting jailbreak screenshots without root cause,
- treating prompt filters as primary controls,
- memorizing every framework mapping,
- trying every optional lab before the capstone.

## Minimum viable study path

If you have limited time before class, read:

1. Module 01 overview,
2. Module 05 deep dive,
3. Module 06 deep dive,
4. Module 07 deep dive,
5. BrokenPilot prototype app README,
6. BrokenPilot gold-standard final report example.

## How to use module delivery profiles

Every module has a `delivery-profile.md` file. It tells you what is core, what can be compressed, what becomes lab work, and what should move to self-study.

Students can use these profiles to understand what will be assessed live and what is optional enrichment.
