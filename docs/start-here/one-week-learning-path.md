# One-Week Learning Path

The recommended instructor-led version of this course is a **40-hour / one-week training**.

This path is designed for students who want a serious but practical ML Security learning experience without stretching the material over a semester-style schedule.

## What this path gives you

By the end of the week, you should be able to:

- reason about AI systems using security engineering principles,
- identify ML, LLM, RAG, and agent attack surfaces,
- reproduce selected failures in DVAIA and BrokenPilot,
- design concrete controls,
- validate whether controls actually work,
- write decision-grade findings,
- communicate AI security risk to leadership.

## Weekly structure

| Day | Focus | Main outputs |
|---|---|---|
| Day 1 | Foundations and ML architecture | System context diagram, abuse cases |
| Day 2 | LLM and RAG security | DVAIA/BrokenPilot evidence log, mitigation design |
| Day 3 | Agent security and MLOps | Tool permission matrix or executive risk memo |
| Day 4 | Privacy, adversarial ML, and red teaming | Strong rewritten finding |
| Day 5 | BrokenPilot capstone | Final report and presentation |

## How to study during the week

For each major topic:

1. read the overview,
2. read the deep dive where available,
3. study the attack anatomy,
4. complete the lab or tabletop exercise,
5. fill the template,
6. compare your work with the worked examples,
7. write or improve one finding.

## What to prioritize

If time is limited, prioritize:

- Module 05  -  LLM Application Security,
- Module 06  -  RAG Security,
- Module 07  -  Agent and Tool Security,
- Module 11  -  AI Red Team Methodology,
- Module 12  -  BrokenPilot Capstone.

The foundations in Modules 01–04 are still important, but they support the practical work in the later modules.

## Required lab setup

For the hands-on one-week path, you should have:

- Git,
- Python,
- Docker,
- a browser,
- DVAIA for selected LLM/RAG labs,
- BrokenPilot prototype app for agent/tool and capstone labs.

See the Lab Setup section before the first day.

## Reading expectation

This course is reading-first and lab-supported.

You are not expected to memorize every framework or risk taxonomy. You are expected to understand the security reasoning behind the attack classes and be able to apply that reasoning to realistic systems.
