# References — Module 01: Security Engineering for AI

This module intentionally combines classic security engineering with AI-specific guidance.

## Classic security engineering

- Gary McGraw — *Software Security: Building Security In*
- Ross Anderson — *Security Engineering*
- Adam Shostack — *Threat Modeling: Designing for Security*
- Saltzer and Schroeder — *The Protection of Information in Computer Systems*
- Michael Howard and David LeBlanc — *Writing Secure Code*
- Bruce Schneier — *Secrets and Lies*

## Teaching note

Use these references as anchors, not as a checklist to memorize. The purpose of Module 1 is to build a way of thinking. Later modules map specific risks and labs to OWASP, BIML, NIST, and MITRE ATLAS in more detail.
