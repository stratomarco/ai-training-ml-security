## Reading guidance

Students should use OWASP ML Top 10 as a practitioner taxonomy, not as a complete security program.

For deeper security engineering, connect each OWASP ML category back to:

- Asset identification
- Trust boundaries
- Abuse cases
- Access control
- Data integrity
- Privacy
- Supply chain integrity
- Monitoring
- Incident response
- Residual risk

## Instructor note

The OWASP ML Top 10 is useful for structure, but the course should avoid treating it as exhaustive. Use NIST's adversarial ML taxonomy and MITRE ATLAS to broaden the discussion when students ask for more formal terminology or adversarial technique mapping.
