# Weak Risk Register Example  -  BrokenPilot

| Risk | Severity | Mitigation |
|---|---|---|
| Prompt injection | High | Better prompt |
| Data leak | High | Access control |
| Agent problem | Medium | Monitoring |

## Why this is weak

Problems:

- Risks are not specific.
- There is no scenario.
- There is no owner.
- Mitigations are vague.
- There is no residual risk.
- It cannot drive engineering work.

A strong risk register should be specific enough to become a remediation backlog.
